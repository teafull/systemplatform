# 软件平台架构设计文档

## 1. 架构概述

本平台采用**组件化架构**，支持：
- **独立组件开发**：每个组件包含独立的前端和后端
- **前端页面集成**：平台主应用集成各组件的前端页面
- **单点登录（SSO）**：所有组件共享统一认证体系
- **统一授权**：所有组件都需要经过授权中心的权限验证

## 3. 核心模块

### 3.1 认证中心（SSO）

**职责**：
- 用户统一登录/登出
- Token颁发和验证
- 用户管理
- 会话同步

**技术选型**：
- 后端：Node.js + Express + JWT
- 前端：React
- 存储：Redis（会话）+ PostgreSQL（用户数据）

**端口**：4000

### 3.2 授权中心（AuthZ）

**职责**：
- RBAC权限模型管理
- 组件级授权控制
- 资源级授权检查
- 角色和权限管理
- Casbin策略管理

**技术选型**：
- 后端：Node.js + Express
- 权限引擎：Casbin
- 存储：PostgreSQL

**权限模型**：
- `none` - 无访问权限
- `read` - 只读访问
- `write` - 读写访问
- `admin` - 管理员权限

**端口**：6000

### 3.3 组件注册中心

**职责**：
- 组件信息注册和管理
- 组件动态加载控制
- 组件路由配置
- 组件生命周期管理

**数据结构**：
```typescript
interface ComponentInfo {
  id: string;              // 组件唯一标识
  name: string;            // 组件名称
  version: string;         // 版本号
  frontendUrl: string;     // 前端资源地址
  backendUrl: string;      // 后端API地址
  routePath: string;       // 路由路径
  icon?: string;           // 组件图标
  description?: string;    // 组件描述
  enabled: boolean;        // 是否启用
  dependencies?: string[]; // 依赖的其他组件
}
```

**端口**：5000

### 3.4 前端集成框架（微前端）

**职责**：
- 主应用框架搭建
- 子组件动态加载
- 组件间通信
- 路由管理
- 样式隔离

**技术选型**：
- 主框架：React + qiankun
- 路由：React Router
- 状态管理：Zustand
- 通信：自定义事件 + 共享状态

## 4. 组件化开发规范

### 4.1 组件目录结构

```
components/
├── component-a/                    # 组件A
│   ├── frontend/                   # 前端工程
│   │   ├── src/
│   │   ├── package.json
│   │   └── build/                  # 构建产物
│   ├── backend/                    # 后端工程
│   │   ├── src/
│   │   ├── package.json
│   │   └── config/                 # 配置文件
│   ├── manifest.json               # 组件清单（元数据）
│   └── README.md                   # 组件说明
├── component-b/                    # 组件B
│   └── ...
```

### 4.2 组件清单（manifest.json）

```json
{
  "id": "component-a",
  "name": "示例组件A",
  "version": "1.0.0",
  "entry": "//localhost:3001/index.js",
  "container": "#subapp-container",
  "activeRule": "/component-a",
  "backendUrl": "http://localhost:3002",
  "permissions": ["user.read", "user.write"],
  "dependencies": []
}
```

### 4.3 组件间通信机制

**1. 父子通信**（主应用 ↔ 子应用）
- Props传递
- 自定义事件

**2. 子应用间通信**
- 全局状态管理（Redux）
- 自定义事件总线
- 共享存储（LocalStorage + 命名空间）

**3. 后端通信**
- 统一API网关
- 服务间直接调用
- 消息队列

## 5. 单点登录流程

### 5.1 登录流程

```
1. 用户访问平台主应用
   ↓
2. 主应用检查Token（LocalStorage）
   ↓
3. Token无效 → 重定向到认证中心
   ↓
4. 用户在认证中心登录
   ↓
5. 认证中心颁发JWT Token
   ↓
6. 重定向回原应用（带Token）
   ↓
7. 主应用存储Token并初始化用户信息
   ↓
8. 加载启用的组件
   ↓
9. 各子应用通过共享Token访问各自后端
```

### 5.2 登出流程

```
1. 用户点击登出
   ↓
2. 主应用清除本地Token
   ↓
3. 通知所有子应用清理状态
   ↓
4. 调用认证中心登出接口
   ↓
5. 清除所有子应用的会话
   ↓
6. 重定向到登录页
```

## 6. 技术栈总览

| 模块 | 技术 | 说明 |
|------|------|------|
| 主应用前端 | React 18 + TypeScript | 主框架 |
| 微前端框架 | qiankun 2.x | 子应用加载 |

| 缓存 | Redis | 会话缓存 |

| API网关 | Nginx/Kong | 统一入口 |

## 7. 安全机制

### 7.1 认证安全
- JWT Token + RefreshToken双token机制
- Token过期自动刷新
- HTTPS加密传输
- XSS/CSRF防护

### 7.2 授权机制
- **RBAC权限模型**：基于角色的访问控制
- **组件级授权**：控制用户对组件的访问（none/read/write/admin）
- **资源级授权**：细粒度的资源操作权限
- **Casbin策略**：灵活的权限策略管理
- **租户隔离**：支持多租户的权限隔离
- **权限继承**：角色权限继承和组合

### 7.3 数据隔离
- 组件数据隔离
- 用户数据隔离
- 租户数据隔离


## 9. 扩展性设计

### 9.2 插件化扩展
- 支持第三方组件注册
- 组件市场机制
- 组件版本管理
